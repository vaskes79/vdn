# Правила проекта VDN

## Структура документации

### Расположение документации

- Вся документация по проекту хранится в директории `docs/`
- Файлы документации используют формат Markdown (`.md`)

### README.md

- Файл `README.md` используется как навигация по папке `docs/`
- В нем содержится:
  - Минимальное описание проекта
  - Инструкции по запуску проекта
  - Все ссылки на разделы документации

### Архитектура проекта

- Файл с описанием архитектуры проекта хранится в `docs/architecture.md`
- В нем фиксируются все архитектурные нюансы проекта
- При внесении архитектурных изменений в проект, они должны быть зафиксированы в файле архитектуры

### Пользовательские истории

- Пользовательские истории хранятся в директории `docs/userstorys/`
- Каждый файл в этой директории описывает способ взаимодействия пользователей с приложением
- Формат имени файла: `название-истории.md` (например, `add-video.md`)

## Структура проекта

### Папка src_backup

- В папке `src_backup` хранится старая версия проекта на время имплементации новой версии приложения
- Эта папка используется как резервная копия рабочего кода во время рефакторинга или переписывания приложения

## Правила работы с документацией

1. При добавлении нового функционала необходимо обновить соответствующую документацию
2. Архитектурные изменения должны быть отражены в `docs/architecture.md`
3. Новые пользовательские сценарии должны быть задокументированы в `docs/userstorys/`
4. README.md должен содержать актуальные ссылки на всю документацию
